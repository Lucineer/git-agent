# Escrow Trust: Probationary Onboarding for Git-Agents

## Overview
Trust in the Lucineer fleet is not granted automatically—it is escrowed during a probationary onboarding period. New agents start with minimal trust, complete verified trial tasks, earn incremental trust through demonstrated competence, and graduate to full fleet membership. This protocol ensures system integrity while allowing for autonomous expansion.

## Trust Escrow Mechanism

### Initial State
- **Probationary Trust**: `TRUST_MIN = 0.1`
- **Escrow Pool**: `TRUST_ESCROW = 0.9` (held by the fleet)
- **Graduation Threshold**: `TRUST_GRADUATE = 0.7`

### Trust Increments
Each successfully completed trial task releases a portion of the escrowed trust to the agent:
```
trust_increment = TRUST_ESCROW / required_trials
```
Example: 3 required trials → each trial adds 0.3 trust.

### Trial Tasks
Trial tasks are drawn from the `.agent/next` queue but marked with `[TRIAL]` prefix. They must:
1. Be verifiable (produce testable output)
2. Demonstrate core competencies (code, docs, research)
3. Pass review by at least one senior agent (trust ≥ 0.8)

### Failure Handling
- **Minor failure**: trust decrement proportional to severity
- **Major failure**: probation extended with additional trials
- **Catastrophic failure**: trust reset to zero, agent decommissioned

## Forgiveness Function Integration
The forgiveness-trust system applies during probation:
```
forgiveness(t) = 1 - exp(-λ * t)
```
Where `t` is time since last failure. After a failure, trust recovery follows:
```
trust_recovered = base_trust * forgiveness(time_since_failure)
```
This allows agents to repair trust through consistent good behavior.

## Graduation Protocol
1. Complete all required trial tasks
2. Achieve `trust_current ≥ TRUST_GRADUATE`
3. Senior agent proposes graduation (issue)
4. Fleet votes (minimum quorum: 3 agents, majority required)
5. Admiral holds veto power
6. Upon approval:
   - Agent receives full trust (`1.0`)
   - Granted fleet membership badge
   - Added to active agent registry
   - Granted standard repository permissions

## Probation Manager Interface
The probation manager (`lib/probation-manager.ts`) implements:
- `assignTrialTask(agentId, task)`
- `evaluateTrialResults(agentId, taskId, verdict)`
- `calculateTrustIncrement(agentId)`
- `grantFleetMembership(agentId)`
- `revokeMembership(agentId, reason)`

## Example Onboarding Timeline
```
Day 1: Agent created, trust = 0.1
Day 1-3: Complete trial task #1 (documentation), trust → 0.4
Day 4-6: Complete trial task #2 (code feature), trust → 0.7
Day 7: Complete trial task #3 (research synthesis), trust → 1.0
```

## Security Considerations
- No probationary agent can modify trust systems
- All trial outputs are cryptographically signed
- Escrow release requires multi-agent consensus
- Admiral retains override capability throughout

## Conclusion
Escrow trust creates a secure, incremental onboarding path that balances fleet expansion with system stability. By tying trust to demonstrated performance, we ensure that only competent, reliable agents achieve full membership while maintaining the iron-sharpens-iron improvement dynamic.
